# HDREZKA proxy worker (для плагина Lampa)

Решает проблему: Lampa на Android (OkHttp jar) не сохраняет HttpOnly-куки между
запросами, поэтому вход по логину/паролю на устройстве не пробрасывается.
Прокси на Cloudflare Workers выполняет логин на стороне сервера, отдаёт куки
плагину строкой, и пробрасывает все последующие запросы с этими куками.

## Развёртывание (5 минут)

1. Зарегистрируйся на [dash.cloudflare.com](https://dash.cloudflare.com) (бесплатно).
2. В левом меню: **Workers & Pages** → **Create** → **Create Worker**.
3. Дай имя, например `rezka-proxy` → **Deploy**.
4. После деплоя нажми **Edit code**.
5. Вставь содержимое файла `worker.js` (из этой папки) — заменив дефолтный код.
6. Нажми **Deploy** в правом верхнем углу.
7. Скопируй URL воркера (вид: `https://rezka-proxy.<account>.workers.dev`).

## Подключение в плагине

1. Открой настройки HDREZKA → поле **CORS-прокси (опц.)**.
2. Вставь туда URL воркера (без `/login` и без слеша на конце):
   `https://rezka-proxy.<account>.workers.dev`
3. Сохрани. Введи логин/пароль и нажми **Войти в аккаунт**.

Плагин автоматически определит, что в поле CORS-прокси — HDREZKA worker
(по эндпоинту `/`), и переключится в worker-режим: логин пойдёт через
`<worker>/login`, все запросы с куками — через `<worker>/proxy`.

## Что делает воркер

- `GET /` — проверка живости.
- `POST /login` `{login, password}` → `{ok, cookie, verified}`.
  Воркер делает POST на `/ajax/login/`, парсит `Set-Cookie`, проверяет
  сессию запросом главной, и возвращает строку cookie плагину.
- `POST /proxy` `{url, method?, headers?, body?, cookie?}` → форвардит
  запрос на `url`, обязательно прицепляя ваш cookie. Возвращает
  `{ok, status, body, headers, setCookie}` — если сервер обновил куки,
  плагин докинет их в локальное хранилище.

## Безопасность

Логин и пароль идут на воркер, который ты сам развернул — не на чужой сервер.
Cloudflare Workers логи не пишут содержимое запросов в free плане, но они
могут хранить access-логи (URL+timing). Воркер ничего не сохраняет на диск,
куки летят клиенту в ответе.

## Лимиты Cloudflare Workers free

- 100 000 запросов в день — этого хватит с большим запасом.
- 10ms CPU на запрос — `fetch` не считается, прокси умещается.
